# 提案：连接 Bilibili（一键启用采集）

## 背景

当前 B 站采集 MVP 已具备端到端能力（浏览器侧主动触发采集 → PineSnap 服务端接收并落库）。但若要求用户理解并手动管理 “Capture Token / Userscript” 概念，会显著提高真实用户的使用门槛。

## 目标

- PineSnap SHALL 提供一个面向普通用户的“连接 Bilibili”入口与说明，使用户无需理解 Userscript/token 概念即可完成启用。
- 用户在 PineSnap 登录态下完成“一键启用”后，系统 SHALL 生成并管理跨站采集所需的授权凭证，并引导用户完成浏览器侧安装。
- 系统 MUST 保留撤销能力：用户可在 PineSnap 中断开连接，使后续采集请求立刻失效。

## 非目标

- 不实现：采集页（贴链接/上传文件等统一采集入口）的最终产品形态与信息架构；本变更仅提供临时入口（右上角菜单 + 连接页）。
- 不实现：Chrome Web Store 等浏览器扩展发布流程。
- 不实现：多平台连接（YouTube 等）；但设计应允许未来扩展。

## 产出物

- 新增连接页：`/connect/bilibili`（说明 + 一键启用）
- 新增安装端点：返回个性化安装脚本（自动内置 PineSnap URL 与授权信息）
- 更新右上角入口文案与引导文档
